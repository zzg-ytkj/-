GitHub Actions 分析流水线
自托管 Runner 监听 push 和 issue 事件，自动运行分析脚本并将结果回写到仓库。
1

创建 GitHub Actions Workflow 文件

在每个学生仓库中创建 .github/workflows/digital-twin.yml，配置自托管 Runner 触发条件。

name: 数字孪生分析引擎 on: push: branches: [main] schedule: - cron: '0 */4 * * *' # 每4小时自动运行 repository_dispatch: types: [analyze] jobs: analyze: runs-on: self-hosted steps: - uses: actions/checkout@v4 - name: 运行分析脚本 run: node analyze.js - name: 回写分析结果 run: | # 通过 GitHub API 更新 Issue 预警 # 通过 PR 评论推送成长曲线 # 更新 Projects 看板健康度字段
2

部署自托管 Runner

在一台校内服务器上部署 Runner，连接到 GitHub Actions。Runner 负责拉取仓库数据、运行分析脚本、回写结果。

# 在校内服务器上： # 1. 创建 Runner 目录 mkdir actions-runner && cd actions-runner # 2. 下载 Runner curl -o actions-runner.sh https://.githubusercontent.com/actions/runner/releases/download/v2.316.0/actions-runner-linux-arm64-2.316.0.tar.gz # 3. 配置 Runner（填入 repo 和 token） ./config.sh --url https://github.com/YOUR_ORG/YOUR_REPO --token XXXXX # 4. 启动 Runner ./run.sh
3

创建分析脚本 analyze.js

分析脚本从 GitHub API 拉取 commit 序列，对文本做能力维度打分，回写 Issue 预警。

// analyze.js - 数字孪生分析引擎核心逻辑 const { Octokit } = require('@octokit/rest'); const { Octokit: GQL } = require('@octokit/graphql'); const octokit = new Octokit({ auth: process.env.GH_TOKEN }); // 1. 拉取 commit 序列 const { data: commits } = await octokit.repos.listCommits({ owner: OWNER, repo: REPO, per_page: 100 }); // 2. 时序分析 - 检测写作停滞 const gaps = detectGaps(commits); // 连续5天无提交 → 创建预警 Issue if (gaps.length > 0) { await octokit.issues.create({ owner: OWNER, repo: REPO, title: `⚠️ [数字孪生] 写作停滞预警 - ${studentName}`, body: `学生已连续 ${gaps.days} 天无有效提交，建议主动联系。`, labels: ['digital-twin', 'warning'] }); } // 3. 能力维度打分（调用校内大模型） const score = await analyzeCapability(text); // → logic_coherence: 逻辑连贯度 // → argument_depth: 论证深度 // → innovation_clarity: 创新点明确度 // 4. 回写能力曲线到 PR 评论 await octokit.rest.issues.createComment({ owner: OWNER, repo: REPO, issue_number: PR_NUM, body: generateGrowthReport(score) });

一键部署指南

粗粒度数字孪生平台基于 GitHub 原生数据运行，无需在学生电脑安装任何客户端。

第一步：获取 GitHub Personal Access Token

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token

需要的权限：repo（所有仓库）、read:org（组织信息）、read:project（Projects）

第二步：组织或用户

在 GitHub 上创建一个组织（如 yantai-tech-thesis），邀请所有学生加入。每个学生的仓库命名格式：thesis-张三、thesis-李四。

第三步：在平台填入配置

左侧配置面板：填入 Token → 组织名 → 仓库前缀（thesis-）→ 点击"连接 GitHub"

平台将自动完成：

  • 扫描组织下所有 thesis-* 仓库
  • 拉取每个仓库的 commit 序列、Issue、Projects 数据
  • 分析写作节奏、检测停滞预警、生成成长曲线
  • 展示群体洞察（哪些章节是共性难点）

重要提示：平台读取的是 GitHub 公开的提交历史和项目管理数据。如需更深度分析（文本内容分析），需额外部署分析脚本（见"部署流水线"页面）。
